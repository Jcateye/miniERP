# DFP 数据库模型基线冻结（v1.1）

## 目标
冻结逻辑模型、物理类型、约束与关键索引，保障后续实现可并行且一致。

## 1. 命名与主键规范（强制）
- 表名统一单数：`idempotency_record`, `outbox_event`
- 主键统一：`id bigint generated by default as identity`
- 所有外键字段与主键类型保持一致（均 `bigint`）

## 2. 核心实体分组
### 2.1 租户与权限
- `tenant`, `user`, `role`, `permission`, `user_role`, `api_client`, `api_call_log`

### 2.2 主数据
- `sku`, `sku_mapping`, `sku_substitution`, `warehouse`

### 2.3 采购/入库
- `purchase_order`, `purchase_order_line`, `grn`, `grn_line`

### 2.4 销售/出库
- `sales_order`, `sales_order_line`, `outbound`, `outbound_line`

### 2.5 库存/盘点
- `inventory_ledger`, `inventory_balance`, `stocktake`, `stocktake_line`

### 2.6 报价
- `quotation`, `quotation_version`, `quotation_line`

### 2.7 证据
- `evidence_asset`, `evidence_link`

### 2.8 横切能力
- `audit_log`, `state_transition_log`, `idempotency_record`, `outbox_event`

## 3. 关系基线
- 单据头 `1-N` 单据行
- 单据 `1-N` 状态迁移日志
- 证据资产与业务实体通过 `evidence_link` 解耦（支持 document/line）

## 4. 关键约束（tenant 维度 + 业务一致性）
### 4.1 唯一约束
- `(tenant_id, doc_no)`
- `(tenant_id, sku_code)`
- `(tenant_id, idempotency_key)` on `idempotency_record`
- 证据去重：`(tenant_id, asset_id, entity_type, entity_id, scope, coalesce(line_ref, 0))`

### 4.2 CHECK 约束
- `inventory_balance.on_hand >= 0`
- `evidence_link`: `scope='document' => line_ref is null`；`scope='line' => line_ref is not null`

### 4.3 不可变规则
- `inventory_ledger` 禁止 update/delete（仅追加 + 冲销）
- 冲销链路字段：`reversal_of_ledger_id bigint null`（外键指向 `inventory_ledger.id`）

## 5. 索引基线
### 5.1 热路径索引
- `(tenant_id, status, doc_date)`
- `(tenant_id, sku_id, posted_at)` on `inventory_ledger`
- `(tenant_id, entity_type, entity_id, line_ref)` on `evidence_link`

### 5.2 必备补充索引
- 所有外键列必须建索引（头行表、日志关联表、link 表）
- 软删常用查询补充部分索引：`where deleted_at is null`
- RLS 场景 tenant 前缀索引：tenant-owned 表至少一个 `(tenant_id, ...)`

## 6. 物理类型与精度规范
- `id`/外键：`bigint`
- 时间：`timestamptz`
- 金额/数量：`numeric(20,6)`（可按业务扩展到 `numeric(20,8)`）
- 大文本：`text`
- 可枚举状态：`varchar(32)`（后续可迁移 enum）

## 7. RLS 与角色安全（落地要求）
- tenant-owned 表必须 `ENABLE ROW LEVEL SECURITY`
- 生产环境业务角色禁止 `BYPASSRLS`
- 每请求事务必须设置 tenant 上下文（未设置 fail-closed）

## 8. 迁移原则
- 优先前向兼容迁移
- 破坏性字段变更必须先弃用再删除
- 索引与约束变更需附带回滚策略
- 大表（`inventory_ledger`, `audit_log`, `outbox_event`）需定义归档策略
