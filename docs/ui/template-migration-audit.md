# Template Migration Audit

仅覆盖 `apps/web/src/app/(dashboard)` 下本次治理范围：首页、`/settings`、`/reports`。

| 页面 | 当前布局 | 目标模板 | 假按钮 | 修复动作 |
| --- | --- | --- | --- | --- |
| `/` 首页 | 手写概览页，自定义 KPI + 待办 + 快捷入口 + 时间线；未走模板装配 | `T1` | `查看全部`、`完整日志` 为仅有样式的文本按钮 | 切换到 `dashboardOverviewConfig + OverviewAssembly`；头部主操作改为真实入口：`新建 GRN`、`发起 OUT`、`发起盘点`；移除无行为文本按钮 |
| `/settings` | 手写分组入口网格页；未走模板装配 | `T1` | 无显式假按钮，但缺少统一主操作治理 | 切换到 `settingsOverviewConfig + OverviewAssembly`；头部主操作接到真实页面：`主数据配置`、`用户管理`、`系统配置`；快捷入口补齐到真实设置子页 |
| `/reports` | 手写 KPI + 报表分类卡片；分类卡片跳转静态占位详情，缺少真实后端能力判断 | `T1` | 5 张报表分类卡片均为假入口；缺少受控主操作 | 切换到 `reportsOverviewConfig + OverviewAssembly`；头部保留真实来源数据入口：`查看库存流水`、`采购来源数据`；无后端能力的导出与报表分类统一 `disabled`，并在文案中标明“后端未开通/未接入” |

## 按钮治理规则

1. 有真实行为即保留入口：可导航到已存在且可执行的页面，或已接入 BFF / 后端能力。
2. 无真实后端能力即禁用：不再允许跳转到静态占位页冒充可用功能。
3. 禁用按钮必须显式写明原因，例如 `导出报表（后端未开通）`、`库存价值报表（后端未接入）`。
