## 3. 分阶段路线图

## Phase 1（W1-W6）：可用性止血 + 工程基线强执行

### 目标
消除“不可用感”（布局混乱、假按钮、KPI混乱），建立统一研发门禁。

### 关键交付
1. 页面模板收敛（双体系收敛到 T1/T2/T3/T4）
2. 假按钮清零（动作按钮必须可执行）
3. KPI治理（数据来源/更新时间/跳转去向明确）
4. PR门禁上线（模板合规+状态契约+过账一致性）

### 里程碑
- M1（W2）：关键路径页面无“点了没反应”
- M2（W4）：同类页面布局一致率 >= 90%
- M3（W6）：门禁稳定运行，新增页面100%模板合规

### 验收指标
- 假按钮数量：0（关键路径）
- T模板覆盖率：>=90%
- 关键页面可用性冒烟：100%通过

## Phase 2（W7-W16）：Inventory Lite 独立可售卖

### 目标
交付独立库存模块，不依赖采购/销售也能完整运行。

### 功能范围（P0）
- 库存台账（balances）
- 库存流水（ledger）
- 单据：IN / OUT / TRANSFER / ADJ
- 流程：创建 -> 校验 -> 过账 -> 冲销
- 审计：who/when/what/request_id
- 主数据：SKU / 仓库（库位预留）

### 技术关键点
- `inventory_ledger` 单一事实源
- 幂等键去重 + payload hash 校验
- 文档级+SKU级并发控制
- 事务原子提交 + 反向流水补偿

### 里程碑
- M4（W10）：台账/流水真数据上线
- M5（W13）：四类单据闭环可跑
- M6（W16）：并发/幂等/冲销验收通过，达到试点交付

### 验收指标
- 重复提交100次仅1次成功过账
- 并发出库无负库存
- 冲销后库存回补一致且可审计
- 客户试点可连续使用

---

## Phase 3（M5-M12）：一体化ERP扩展

### 目标
从库存单点能力扩展为可竞争的业务闭环产品。

### 模块扩展顺序
1. 采购增强：PO/GRN 创建、审批、到货差异、对账
2. 销售增强：报价 -> SO -> OUT 正式链路
3. 财务衔接：应收应付、收付款（轻量版）
4. 报表闭环：业务页直达经营指标与明细追溯
5. 本地化深化：税票/合规/审计增强

### 年度指标（建议）
- 核心链路成功率 >= 99%
- 新功能模板合规率 = 100%
- 核心命令 P95 满足SLO
- 客户续费/增购率持续提升

## 4. 组织与治理

### 4.1 组织分工
- 产品线A：库存与仓储
- 产品线B：采购与销售
- 平台线：BFF/权限/审计/可观测
- 体验治理线：模板、设计一致性、可用性门禁

### 4.2 节奏机制
- 每周：项目推进会（风险、阻塞、偏差）
- 每两周：里程碑评审与演示
- 每月：架构审计 + 体验审计

### 4.3 变更控制
- 任何触碰工程红线的需求必须先评审
- 任何与ADR冲突的实现必须先修订ADR

## 5. 风险与应对

1. **壳页面继续扩张**
   - 应对：冻结新增页面窗口，先补全关键交互
2. **状态契约漂移**
   - 应对：`packages/shared` 单一出口 + CI校验
3. **库存并发问题**
   - 应对：锁顺序规范、死锁重试、并发压测
4. **客户预期与交付偏差**
   - 应对：试点客户双周同步 + 演示验收机制
