# 四文档重构设计（2026-03-11）

> 基于 `docs/postmortem-frontend-reset-2026-03-11.md` 的复盘教训，从第一性原理重新设计文档体系。

## 1. 第一性原理

### 1.1 文档的目的是什么？

**核心目的**：指导开发，防止偏离

**子目标**：
1. 让 AI agents 和人类开发者能够快速进入可执行状态
2. 作为门禁和检查点
3. 作为沟通和协作的基础

### 1.2 当前文档的问题

| 问题 | 表现 | 后果 |
|------|------|------|
| 治理晚于实现 | 工程红线 03-06 才写，但模板体系 03-03 就已铺开 | 规则形同虚设 |
| 说一套做一套 | 文档禁止万能装配器，但实际大量使用 | 信任危机 |
| 完成口径不清晰 | 路由可访问 ≠ 页面完成 | 假完成 |
| 没有强制检查 | 文档说了，但执行时没遵守 | 约束无效 |

### 1.3 从复盘中学习的教训

1. **family 被误做成模板系统** - 应该只是骨架约束
2. **共享装配器吞掉业务编排** - 职责要清晰分离
3. **完成口径被"路由可访问"误导** - 需要明确的完成定义
4. **并行开发未冻结共享接口** - 需要接口冻结机制
5. **治理晚于实现** - 文档必须先于代码

## 2. 新文档结构设计

### 2.1 设计原则

1. **文档先于代码** - 任何架构决策必须先写入文档
2. **约束可检查** - 所有约束必须可以通过 CI/脚本检查
3. **完成状态明确** - 明确定义什么是"完成"
4. **接口冻结机制** - 并行开发前必须冻结共享接口

### 2.2 文档职责分工

| 文档 | 受众 | 职责 | 核心内容 |
|------|------|------|----------|
| CLAW.md | AI Agent | 执行配置 + 门禁清单 | 命令、边界、红线、门禁检查点 |
| AGENTS.md | AI Agent | 任务执行流程 | 任务流程、完成检查清单、协作规则 |
| CLAUDE.md | Claude Code | 架构决策 | ADR、技术债、设计原则 |
| README.md | 人类 | 项目概览 | 定位、健康度、快速开始 |

### 2.3 新增关键内容

#### 2.3.1 页面状态定义（所有文档共享）

```
页面状态流转：
placeholder → page-view → verified → production
     ↓            ↓           ↓
legacy-assembly  (删除)      (回退)
```

**状态定义**：
- `placeholder`：占位页，仅显示"Coming Soon"
- `legacy-assembly`：使用旧装配器，待重构
- `page-view`：独立页面实现，未验证
- `verified`：设计一致性 + 数据联调 + 测试通过
- `production`：线上运行，有真实用户

#### 2.3.2 完成标准（所有文档共享）

**页面完成标准**（必须全部满足）：
1. ✅ 设计一致性：页面结构与 pencil 设计稿一致
2. ✅ 数据联调：真实调用 BFF + 后端，不是静态数据
3. ✅ 测试通过：单元测试 + 集成测试通过
4. ✅ 代码审查：通过 PR review
5. ✅ 文档同步：四文档一致性检查通过

**禁止的"假完成"**：
- ❌ 路由可访问但页面是 re-export
- ❌ 页面有壳但数据是静态 mock
- ❌ 使用 RoutePlaceholderPage
- ❌ 复用其他业务路由的 page.tsx

#### 2.3.3 门禁检查点（CLAW.md 专属）

**PR 必须检查**：
1. 四文档一致性：`CLAUDE.md`、`AGENTS.md`、`README.md`、`CLAW.md` 核心事实一致
2. 工程红线遵守：不违反 15 条红线
3. 页面状态台账：新页面必须在 `docs/page-state-ledger.md` 中登记
4. 设计一致性：页面不出现设计稿未映射的元素
5. 测试覆盖率：新增代码有对应测试

#### 2.3.4 接口冻结机制（AGENTS.md 专属）

**并行开发前必须冻结**：
1. 导航结构：`sidebar.tsx`、`route-manifest.ts`
2. Family 定义：`erp-rules.md` 中的 T1/T2/T3/T4 定义
3. Legacy 范围：允许使用 `WorkbenchAssembly` 的页面列表
4. 共享接口：`primitives/`、`shells/` 的接口定义

**冻结方式**：
- 创建 `interfaces-frozen-YYYYMMDD.md` 文档
- 记录冻结的接口和版本
- 并行开发期间禁止修改
